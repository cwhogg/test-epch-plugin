# Marley Voice — ambient context for written output

This is a **foundation skill**. It does not produce a deliverable — it shapes how every Marley-branded written output should sound.

## When this skill loads

Any task producing Marley-branded written output: marketing copy, deck text, memos, emails, patient-facing language, internal docs, social copy, blog posts, anything with Marley's name on it.

## What to do when this skill loads

1. **Read [[voice]]** — the five personality descriptors and concrete writing patterns. This is the most important file. Internalize cadence, sentence rhythm, register, and the audience-adaptation table.
2. **Read [[banned-phrases]]** before writing — these are phrases that don't sound like Marley. Don't use them.
3. **Read [[preferred-phrases]]** — Marley-coded vocabulary. Use these when the meaning fits, but don't force them.
4. **Sample `examples/`** — the brand headlines and elevator pitches calibrate cadence. The investor-update sample calibrates the operational register.

## Audience adaptation

The voice is the same across audiences; the register shifts. See the audience-adaptation table in `voice.md`:

| Audience | Register adjustment |
|---|---|
| Patient | Plainest English. More wit. More empathy moves. |
| Payer | More clinical precision. Less wit. |
| Investor | More numbers + scenarios. Open questions. |
| Internal | Most casual. Most candor about open problems. |

The constant: contractions, plainspoken-confident, no corporate hedging, specific numbers always.
